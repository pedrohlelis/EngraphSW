# UI System Skill

## Purpose

Maintain a consistent professional engineering-tool visual language.

---

# Design Direction

The interface should feel similar to:
- Claude Code
- Linear
- Raycast
- Vercel

---

# Visual Philosophy

Prioritize:
- clarity
- focus
- graph readability
- information density
- professional engineering aesthetics

Avoid:
- excessive gradients
- colorful dashboards
- marketing-heavy SaaS visuals
- visual clutter

---

# Theme Direction

Use:
- dark-first design
- restrained color palette
- warm orange accents
- muted surfaces
- subtle shadows

Suggested palette:
- near-black backgrounds
- charcoal surfaces
- muted gray borders
- warm typography
- amber/orange highlights

---

# Node Styling

Nodes should emphasize:
- readability
- execution state visibility
- dependency visibility
- structured hierarchy

---

# Component Rules

Favor:
- reusable primitives
- consistent spacing
- restrained border radius
- subtle depth

Avoid:
- inconsistent Tailwind classes
- one-off visual patterns
- random color usage

---

# UX Philosophy

The platform should feel like:
- a professional engineering operating system
- an advanced developer tool
- a graph-native engineering workspace
