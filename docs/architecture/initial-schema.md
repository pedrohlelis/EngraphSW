## Purpose

This document defines the logical Prisma schema for EngraphSW. It is not a `.prisma` file but is precise enough for direct translation to one.

All field names, types, constraints, and relationships are normative. The Implementer reads this document and scaffolds `prisma/schema.prisma` without making schema design decisions.

## Conventions

### Soft Deletion

All artifact tables carry:
- `deleted_at DateTime?` — null for active records; set to deletion timestamp for soft-deleted records
- All queries filter `WHERE deleted_at IS NULL` unless explicitly querying history

### Timestamps

All tables carry:
- `created_at DateTime @default(now())`
- `updated_at DateTime @updatedAt`

Audit-trail tables (execution logs, outbox, snapshots) carry `created_at` only — they are append-only and never updated.

### IDs

All tables use `String @id @default(cuid())` — CUID2 string IDs generated by the application layer. No auto-incrementing integers for domain entities.

### Table Prefixes

Table prefixes follow module ownership (see [Persistence Strategy](persistence-strategy.md) and [Module Conventions](module-conventions.md)):
- `ws_` — workspace module
- `graph_` — graph module
- `node_` — nodes module
- `exec_` — execution module
- `trace_` — traceability module
- `version_` — versioning module
- `ai_` — AI module
- `event_outbox` — event outbox (cross-cutting)

## Graph Layer

### `ws_workspaces`

Workspace entity. The top-level organizational container.

```
ws_workspaces
  id          String    @id @default(cuid())
  name        String
  description String?
  owner_id    String               -- user/team identifier (external auth, not FK)
  deleted_at  DateTime?
  created_at  DateTime  @default(now())
  updated_at  DateTime  @updatedAt

  graphs      graph_graphs[]
```

Constraints:
- `name` is required, not unique globally (workspaces are scoped to owner)
- `owner_id` references the external auth system, not a local table

---

### `graph_graphs`

Graph entity. A workspace contains one or more graphs.

```
graph_graphs
  id           String    @id @default(cuid())
  workspace_id String
  name         String
  description  String?
  deleted_at   DateTime?
  created_at   DateTime  @default(now())
  updated_at   DateTime  @updatedAt

  workspace    ws_workspaces    @relation(fields: [workspace_id], references: [id])
  nodes        node_nodes[]
  edges        graph_edges[]
  snapshots    version_snapshots[]
```

Constraints:
- `workspace_id` required, FK to `ws_workspaces`
- A graph cannot be created in a soft-deleted workspace

---

### `node_nodes`

Node entity. The primary domain entity of the platform.

```
node_nodes
  id              String       @id @default(cuid())
  graph_id        String
  node_type       String                    -- registered plugin type key (e.g., "requirement", "persona")
  content         Json                      -- JSONB artifact content; schema defined by node_type plugin
  position_x      Float        @default(0) -- canvas X position
  position_y      Float        @default(0) -- canvas Y position
  width           Float?                    -- node visual width (null = plugin default)
  height          Float?                    -- node visual height (null = plugin default)
  exec_status     NodeExecStatus @default(IDLE)
  deleted_at      DateTime?
  created_at      DateTime     @default(now())
  updated_at      DateTime     @updatedAt

  graph           graph_graphs             @relation(fields: [graph_id], references: [id])
  outgoing_edges  graph_edges[]            @relation("source_node")
  incoming_edges  graph_edges[]            @relation("target_node")
  exec_logs       exec_execution_logs[]
  stale_records   exec_staleness_records[]

enum NodeExecStatus {
  IDLE
  RUNNING
  SUCCESS
  FAILED
  STALE
}
```

Constraints:
- `graph_id` required, FK to `graph_graphs`
- `node_type` required; must match a registered plugin type at the application layer (not enforced by DB)
- `content` JSONB — structure is validated by the node type plugin contract, not by a DB constraint
- `exec_status` persisted; survives session close/reopen
- `position_x`, `position_y` are canvas coordinates; persisted with the node for workspace restore
- `node_type` is immutable after creation (enforced at application layer)

---

### `graph_edges`

Edge entity. A typed semantic relationship between two nodes.

```
graph_edges
  id          String    @id @default(cuid())
  graph_id    String
  source_id   String
  target_id   String
  edge_type   String              -- typed edge key (e.g., "DERIVED_FROM", "REFINES", "IMPLEMENTS")
  metadata    Json?               -- JSONB optional edge metadata (labels, weights, custom attributes)
  created_at  DateTime  @default(now())
  updated_at  DateTime  @updatedAt

  graph       graph_graphs     @relation(fields: [graph_id], references: [id])
  source      node_nodes       @relation("source_node", fields: [source_id], references: [id])
  target      node_nodes       @relation("target_node", fields: [target_id], references: [id])
```

Constraints:
- `graph_id` required, FK to `graph_graphs`
- `source_id` required, FK to `node_nodes`
- `target_id` required, FK to `node_nodes`
- `source_id != target_id` (no self-edges; enforced at application layer)
- `edge_type` required; validated against `typed-edge-semantics.md` at application layer
- `edge_type` is immutable after creation
- Edges are not soft-deleted; they are hard-deleted when removed (the semantics of an edge do not persist after the relationship is severed)
- Cascade: when a node is soft-deleted, its edges are hard-deleted

Index recommendations:
- `(graph_id, source_id)` — outgoing traversal
- `(graph_id, target_id)` — incoming traversal
- `(graph_id, edge_type)` — edge type filtering

## Execution Layer

### `exec_execution_logs`

Append-only execution history for each node. Never updated or deleted.

```
exec_execution_logs
  id            String          @id @default(cuid())
  node_id       String
  trigger       ExecTrigger                -- what initiated execution
  batch_id      String?                    -- groups executions from the same propagation batch
  input_snapshot Json                      -- snapshot of upstream context at execution time
  output        Json?                      -- produced output (null if failed before output)
  result        ExecResult
  error_message String?                    -- populated on FAILED result
  started_at    DateTime
  completed_at  DateTime?
  created_at    DateTime        @default(now())

  node          node_nodes      @relation(fields: [node_id], references: [id])

enum ExecTrigger {
  USER_MANUAL      -- user explicitly triggered execution
  DEPENDENCY_CHANGE  -- upstream node completed, triggered recalculation
  STALE_RECOVERY   -- user cleared stale state and triggered recompute
  SYSTEM_BATCH     -- bulk execution triggered by system operation
}

enum ExecResult {
  SUCCESS
  FAILED
  CANCELLED
}
```

Constraints:
- `node_id` FK to `node_nodes`
- Append-only — no updates or deletes
- `input_snapshot` is a JSON snapshot of the relevant upstream content at execution time; not a FK reference
- `batch_id` groups logs from a single propagation cascade for audit visibility

---

### `exec_staleness_records`

Tracks when and why a node was marked stale. Audit trail for staleness propagation.

```
exec_staleness_records
  id             String    @id @default(cuid())
  node_id        String
  cause_node_id  String?               -- the upstream node whose change caused this staleness
  cause_edge_id  String?               -- the edge through which staleness propagated
  resolved_at    DateTime?             -- null until stale state is cleared
  created_at     DateTime  @default(now())

  node           node_nodes  @relation(fields: [node_id], references: [id])
```

Constraints:
- Staleness records are created when a node transitions to STALE
- `resolved_at` is set when the node is re-executed or stale state is manually cleared
- `cause_node_id` and `cause_edge_id` enable traceability of the propagation chain

## Versioning Layer

### `version_snapshots`

Graph snapshots for version history. Append-only.

```
version_snapshots
  id            String          @id @default(cuid())
  graph_id      String
  label         String?                    -- human-readable version label
  snapshot_type SnapshotType
  payload       Json                       -- serialized full graph state at snapshot time
  author_id     String?                    -- user who triggered the snapshot
  created_at    DateTime        @default(now())

  graph         graph_graphs    @relation(fields: [graph_id], references: [id])

enum SnapshotType {
  MANUAL       -- user-triggered
  AUTO         -- system-triggered before significant operations
  PRE_EXEC     -- taken before a batch execution run
  MILESTONE    -- explicitly labeled checkpoint
}
```

Constraints:
- Append-only — snapshots are never modified or deleted
- `payload` JSONB captures the complete graph: nodes (with content), edges, and execution state at snapshot time
- `payload` must be self-contained — snapshot restoration must not require querying other tables

## AI Audit Layer

### `ai_audit_records`

Immutable record of every AI provider request. See [AI Architecture](ai-architecture.md).

```
ai_audit_records
  id              String    @id @default(cuid())
  provider        String               -- "anthropic" | "openai"
  model           String               -- model identifier used
  operation_type  String               -- "generation" | "analysis"
  prompt_template String?              -- template key if used
  input_tokens    Int
  output_tokens   Int
  latency_ms      Int
  result          AuditResult
  error_message   String?
  invoking_module String               -- which module called the AI layer
  created_at      DateTime  @default(now())

enum AuditResult {
  SUCCESS
  FAILED
  RATE_LIMITED
}
```

Constraints:
- Append-only
- `invoking_module` is a string tag (e.g., "requirements", "estimation") for observability
- No FK to application entities — audit records are infrastructure, not domain

## Event Outbox

### `event_outbox`

Transactional outbox for reliable domain event delivery. See [Persistence Strategy](persistence-strategy.md) — Outbox Pattern.

```
event_outbox
  id            String          @id @default(cuid())
  module        String                     -- publishing module (e.g., "nodes", "graph")
  event_type    String                     -- event class name (e.g., "NodeCreatedEvent")
  payload       Json                       -- serialized event payload
  retry_count   Int             @default(0)
  error_message String?
  processed_at  DateTime?                  -- null until successfully published
  created_at    DateTime        @default(now())

  @@index([processed_at])
```

Constraints:
- `processed_at IS NULL` records are pending publication
- A background worker processes pending records and marks them with `processed_at` on success
- `retry_count` increments on failure; max retry threshold enforced at application layer
- Consumers must handle duplicate delivery (at-least-once delivery guarantee)

Index: `processed_at` index is critical for the outbox poller query performance.

## JSONB Content Strategy

### When to use typed columns

A field should be a typed column (not JSONB) when it meets any of these criteria:
- It is queried in WHERE clauses (e.g., `exec_status`, `node_type`, `workspace_id`)
- It participates in a JOIN or FK relationship
- It requires a database-level constraint (NOT NULL, unique, FK)
- It is used in ORDER BY or aggregation

### When to use JSONB

A field should be JSONB when:
- It is artifact content (domain-specific, varies by node type)
- Its schema evolves per artifact type without DB migrations
- It is typically read as a whole, not filtered field-by-field
- It is unlikely to be used in WHERE or JOIN conditions

### JSONB in this schema

| Table | JSONB Field | Reason |
|---|---|---|
| `node_nodes` | `content` | Artifact-specific fields vary by node type |
| `graph_edges` | `metadata` | Optional edge annotations, not queried |
| `exec_execution_logs` | `input_snapshot`, `output` | Execution context snapshot, not queried |
| `version_snapshots` | `payload` | Full graph state blob |
| `event_outbox` | `payload` | Serialized event, not queried field-by-field |

## Schema Summary

```
ws_workspaces
  graph_graphs
    node_nodes
      exec_execution_logs
      exec_staleness_records
    graph_edges

version_snapshots         (FK to graph_graphs)
ai_audit_records          (no FK — infrastructure table)
event_outbox              (no FK — outbox table)
```

All modules with additional persistence requirements (e.g., `traceability/` for traversal cache, `collaboration/` for presence anchors) introduce their own prefixed tables as they are implemented in later phases. This document covers only the tables required for Phase 2 (System Architecture) scaffolding.
